# Asqav Integration

[Asqav](https://github.com/asqav/asqav) is an open-source AI governance and compliance tool. It captures cryptographically signed audit trails for every agent action and provides a CI scanner (`asqav-compliance`) that flags governance violations on pull requests.

Use Asqav with CrewAI when you need:

- **Signed audit trails**: Tamper-evident logs of every task execution, tool call, and agent decision
- **Compliance scanning**: Automated governance checks that run in CI alongside your test suite
- **Incident traceability**: Reproduce and verify exactly what an agent did and when

## Setup

<Steps>
  <Step title="Install Asqav with the CrewAI extra">
    ```bash
    pip install "asqav[crewai]"
    ```
  </Step>
  <Step title="Set your signing key">
    Asqav signs every audit record with an Ed25519 key. Generate one and export it:

    ```bash
    asqav keygen --out ~/.asqav/signing.key
    export ASQAV_SIGNING_KEY=~/.asqav/signing.key
    ```

    For CI, store the key as a repository secret and inject it as `ASQAV_SIGNING_KEY`.
  </Step>
  <Step title="Attach the callback hook to your Crew">
    `AsqavCrewHook` is a CrewAI callback that records task lifecycle events.

    ```python
    from crewai import Agent, Crew, Task, Process
    from asqav.extras.crewai import AsqavCrewHook

    hook = AsqavCrewHook(agent_name="my-crew-project")

    researcher = Agent(
        role="Researcher",
        goal="Find and summarize recent AI governance developments",
        backstory="An analyst who tracks regulatory changes in AI policy",
        verbose=True,
    )

    task = Task(
        description="Summarize AI governance news from the past 30 days.",
        expected_output="A bullet-point summary with source references.",
        agent=researcher,
    )

    crew = Crew(
        agents=[researcher],
        tasks=[task],
        process=Process.sequential,
        step_callback=hook.step_callback,
        task_callback=hook.task_callback,
    )

    result = crew.kickoff()
    print(result)
    ```

    After the run, Asqav writes a signed JSONL audit log to `~/.asqav/logs/` by default.
  </Step>
</Steps>

## Audit Log Format

Each record is a JSON line containing the event type, agent role, task description, timestamp, and an Ed25519 signature over the record body:

```json
{
  "event": "task_completed",
  "agent": "Researcher",
  "task": "Summarize AI governance news from the past 30 days.",
  "output_hash": "sha256:a3f1...",
  "timestamp": "2026-04-08T12:00:00Z",
  "signature": "base64:..."
}
```

Verify a log file with:

```bash
asqav verify --log ~/.asqav/logs/my-crew-project.jsonl
```

## CI Governance Scanning

The `asqav-compliance` scanner analyzes agent logs and source code for governance policy violations - things like missing human-in-the-loop gates, unscoped tool permissions, or agents writing to production systems without approval.

Add it to your CI pipeline:

```yaml
# .github/workflows/compliance.yml
name: Compliance

on: [push, pull_request]

jobs:
  asqav-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install "asqav[crewai]"
      - run: asqav-compliance scan --path . --fail-on high
        env:
          ASQAV_SIGNING_KEY: ${{ secrets.ASQAV_SIGNING_KEY }}
```

`--fail-on high` exits non-zero only for high-severity findings, so low-risk warnings do not block merges.
